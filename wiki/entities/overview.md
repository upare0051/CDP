# Alo ActivationOS — Overview

Production-style MVP for a Reverse ETL platform that syncs customer data from data warehouses (Redshift/DuckDB) to marketing platforms (Braze/Attentive).

## Stack

- **Backend:** FastAPI (`backend/app`), entry at [main.py](../../backend/main.py).
  - `adapters/sources/` — DuckDB, Redshift
  - `adapters/destinations/` — Braze, Attentive
  - `api/routes/`, `services/`, `models/`, `schemas/`, `core/`, `db/`
- **Frontend:** React + Vite + TypeScript + Tailwind ([frontend/](../../frontend/)).
- **Metadata DB:** PostgreSQL (SQLite fallback). Stores connections, sync jobs, field mappings, sync runs.
- **Scheduler:** Apache Airflow DAGs at [airflow/dags/](../../airflow/dags/).
- **Transformations:** dbt project at [platform/dbt/](../../platform/dbt/) producing marts:
  - `mart_customer_360`
  - `mart_segment_base`
  - `mart_activation_performance`
  - `mart_ai_customer_insights_team`
  - `mart_ai_audience_recommendations_team`
- **Semantic layer:** Cube ([cube/](../../cube/)).
- **Journeys:** orchestration UI assets at [journeys/](../../journeys/).

## Sync modes

- **Full refresh:** read all rows, send full dataset, no checkpoint.
- **Incremental:** requires `incremental_column` (e.g. `updated_at`); checkpoint = last synced value; only saved on success.

## Warehouse modes

`WAREHOUSE_MODE`:

- `redshift` — production/dev (live)
- `duckdb` — local demo snapshot generated by `scripts/snapshot_redshift_to_duckdb.py` against allowlisted marts.

## Destinations

- **Braze** — `attributes[]` payload keyed by `external_id`.
- **Attentive** — `subscribers[]` payload with `phone`, `email`, `externalId`, `customAttributes`.

## Key config

| Variable | Purpose |
|---|---|
| `WAREHOUSE_MODE` | `redshift` or `duckdb` |
| `DUCKDB_PATH` | Local DuckDB snapshot path |
| `POSTGRES_*` | Metadata DB connection |
| `ENCRYPTION_KEY` | Encrypts stored credentials |
| `MAX_RETRIES` | Max retry attempts for failed syncs (default 3) |
| `SYNC_BATCH_SIZE` | Records per batch (default 1000) |

## Local entry points

- `./start.sh` / `./stop.sh` at repo root
- Backend: `python backend/main.py` → http://localhost:8000 (docs at `/api/docs`)
- Frontend: `npm run dev` in `frontend/` → http://localhost:5173
